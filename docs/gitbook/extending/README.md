# Extending OpenACP

OpenACP is designed to be extended. You can build custom channel adapters that connect OpenACP to any messaging platform, package and distribute those adapters as npm plugins, and contribute improvements back to the core project.

This section covers everything you need to go from zero to a published adapter:

- [Plugin System](plugin-system.md) — Installing, listing, and uninstalling plugins via the CLI.
- [Building Adapters](building-adapters.md) — Step-by-step guide to implementing a `ChannelAdapter`.
- [Adapter Reference](adapter-reference.md) — Full method signatures, type definitions, and lifecycle documentation.
- [Contributing](contributing.md) — Dev setup, testing conventions, and PR process.
