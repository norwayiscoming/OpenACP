# Features

OpenACP comes with powerful features beyond basic chat. These capabilities let you expose local servers to the internet, resume past sessions with full conversation history, track API spending, and diagnose problems — all from your messaging app or terminal.

## What's in this section

- [**Tunneling**](tunnel.md) — Expose local ports to the internet via Cloudflare, ngrok, bore, or Tailscale, with a built-in file viewer for code and diffs.
- [**Context & Resume**](context-resume.md) — Resume sessions with full conversation history from previous agent runs, with built-in history recording and Entire.io checkpoint support.
- [**Output Modes**](output-modes.md) — Choose how much detail you see when your agent is working: just the result, the highlights, or everything.
- [**Usage & Budget**](usage-and-budget.md) — Track token consumption and API costs per session, with configurable monthly budget warnings.
- [**Session Persistence**](session-persistence.md) — Sessions survive daemon restarts; resume exactly where you left off.
- [**Session Handoff**](session-handoff.md) — Transfer a running session from your terminal into Telegram or Discord (and back).
- [**Agent Switch**](agent-switch.md) — Switch between AI agents mid-conversation while preserving context.
- [**Workspaces**](multi-instance.md) — Run separate OpenACP environments for different projects, teams, or use cases.
- [**App Connectivity**](app-connectivity.md) — Connect desktop and web apps to OpenACP via local discovery or remote tunnel links.
- [**Doctor**](doctor.md) — Run a full system health check with `openacp doctor` to catch misconfigurations before they cause problems.
- [**Assistant Mode**](assistant-mode.md) — A dedicated Assistant topic in Telegram that manages your sessions, agents, and settings through natural language.
